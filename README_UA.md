# News Articles Analysis — NLP App

Streamlit-застосунок для аналізу новинних статей за допомогою методів NLP.
Отримує статті з RSS-стрічок обраного видання, витягує повний текст і дає
кілька видів аналізу тексту: n-грами, sentiment, NER, тематичне моделювання,
сумаризацію, частини мови та текстову статистику.

## Структура проєкту

```
news_nlp_app/
├── app.py            # Точка входу: Streamlit UI + диспетчеризація аналізів
├── config.py         # Константи: список джерел новин, список функцій аналізу
├── rss_parser.py      # Завантаження та розбір RSS-стрічки
├── scraper.py         # Отримання повного тексту статті за посиланням
├── analysis.py        # Чисті NLP-обчислення (без UI): sentiment, n-грами,
│                        NER, LDA, сумаризація, POS, textstat
├── visuals.py          # Візуалізація результатів (графіки, таблиці, wordcloud)
└── requirements.txt    # Залежності проєкту
```

Раніше весь код був в одному файлі `app.py` (~600 рядків). Проєкт розбито
на 6 модулів за принципом однієї відповідальності на файл — це спрощує
підтримку та тестування кожної частини окремо.

## Джерела новин

| Джерело        | RSS-стрічка                                       | Статус                        |
|----------------|----------------------------------------------------|--------------------------------|
| NY Times       | `rss.nytimes.com/services/xml/rss/nyt/US.xml`       | ✅ Працює стабільно            |
| Forbes         | `forbes.com/business/feed/`                         | ✅ Працює стабільно            |
| Investing.com  | `investing.com/rss/news_25.rss`                     | ⚠️ Часто недоступний (див. нижче) |

### Відоме обмеження: Investing.com

Investing.com захищений антибот-системою (ймовірно, на кшталт Cloudflare),
яка блокує автоматизовані запити на рівні TLS-з'єднання — навіть із
коректним `User-Agent` заголовком. Це виявляється помилкою
`ConnectionResetError` або порожнім результатом RSS-парсингу.

Застосунок обробляє цю ситуацію без падіння: якщо джерело не повернуло
жодної статті, користувачу показується зрозуміле повідомлення замість
traceback. Для реального доступу до цього джерела знадобився б
headless-браузер або спеціалізована бібліотека (наприклад, `cloudscraper`).

## Встановлення

```bash
pip install -r requirements.txt
```

Знадобиться також модель spaCy для розпізнавання іменованих сутностей:

```bash
python -m spacy download en_core_web_sm
```

Ресурси NLTK (`vader_lexicon`, `averaged_perceptron_tagger`,
`punkt`) довантажуються автоматично при першому використанні відповідних
функцій — вручну встановлювати не потрібно.

## Запуск

```bash
python -m streamlit run app.py
```

## Функції аналізу

- **Intro** — короткий опис обраного новинного джерела
- **Snapshot** — таблиця отриманих статей (заголовок, посилання)
- **Unigrams / Bigrams / Trigrams** — найчастотніші n-грами в заголовках
- **WordCloud** — хмара слів (3 варіанти кольорової схеми)
- **Sentiment Analysis (TextBlob / Vader)** — розподіл тональності заголовків
- **Entity Extraction** — найпоширеніші іменовані сутності (spaCy, NER)
- **Topic Modeling** — тематичне моделювання методом LDA
- **Text Summarization** — автоматична сумаризація тексту статей (LexRank)
- **Parts of Speech** — розподіл частин мови в текстах статей
- **Text Stat** — текстова статистика (читабельність, складність тощо)

## Відомі особливості поведінки

- Sentiment-аналіз рахується за **заголовками**, а не повним текстом статті.
  Заголовки серйозних видань часто написані нейтрально-фактологічно, тому
  результат може показувати переважно клас `neu` — це очікувана поведінка,
  а не помилка.
- Деякі функції (`Topic Modeling`, `Text Summarization`, `Parts of Speech`,
  `Text Stat`) працюють з повним текстом статей (`content`), тому їх якість
  залежить від того, наскільки успішно `scraper.py` зміг витягти текст
  з конкретного сайту.
